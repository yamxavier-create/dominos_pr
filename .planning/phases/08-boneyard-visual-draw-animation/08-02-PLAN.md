2. **BoneyardDrawAnimation.tsx** — Create the animation orchestrator:
   ```
   interface BoneyardDrawAnimationProps {
     myPlayerIndex: number
     playerCount: number
   }
   ```
   - Import `useGameStore` (boneyardDrawQueue, shiftBoneyardDraw, handleBoneyardDraw)
   - Import `useRoomStore` (myPlayerIndex)
   - Import `getPosition` from usePlayerPositions
   - Import `DominoTile` for rendering the flying face-down tile

   Logic:
   - Use `useEffect` watching `boneyardDrawQueue[0]` (the current draw to animate)
   - When a new draw appears at index 0:
     a. Determine the target position using `getPosition(payload.playerIndex, myPlayerIndex, playerCount)` — this gives 'bottom', 'top', 'left', or 'right'
     b. Set local state `animating = true` with the target position
     c. Render a face-down DominoTile (absolute positioned near the pile location — bottom-right of the board area) with the CSS class `boneyard-fly-{position}`
     d. After the animation duration (350ms), call `handleBoneyardDraw(payload, myPlayerIndex)` to apply the actual state change (tile appears in hand)
     e. Call `shiftBoneyardDraw()` to remove the processed draw from the queue
     f. Set `animating = false`
     g. Wait 500ms before the next draw processes (the useEffect will fire again for the new queue[0])

   - Use a ref (`isProcessingRef`) to prevent re-entrant processing
   - The total timing per draw: 350ms animation + 500ms pause = ~850ms per draw
   - Use `setTimeout` chain: start animation -> 350ms later apply state + shift -> 500ms later allow next

   Render: When `animating` is true, render an absolutely-positioned face-down DominoTile (vertical orientation, ~30x60px) with the appropriate `boneyard-fly-{direction}` CSS class. Position the starting point at `bottom: 8px; right: 8px` (matching the BoneyardPile position in GameTable). When not animating, render nothing.
