**1. Create useSpeakingDetection.ts** -- A custom hook that monitors audio levels on MediaStreams and updates callStore.speakingPeers.

Signature:
```typescript
export function useSpeakingDetection(
  remoteStreams: Record<number, MediaStream | null>,
  localStream: MediaStream | null,
  myIndex: number
): void
```

Implementation:
- On mount (and when streams change), create a single `AudioContext`.
- For each non-null stream (remote streams + local stream keyed by myIndex), check for audio tracks (`stream.getAudioTracks().length > 0`). Skip streams with no audio.
- For each valid stream: `audioCtx.createMediaStreamSource(stream)` -> `audioCtx.createAnalyser()` with `fftSize = 256`, `smoothingTimeConstant = 0.5`. Connect source to analyser. Do NOT connect analyser to `audioCtx.destination` (would cause echo).
- Track all created sources in an array for cleanup.
- Poll with `requestAnimationFrame`. For each analyser, call `analyser.getByteFrequencyData(dataArray)`. Take the first 8 bins (speech frequencies), compute average. If average > 25, player is speaking.
- Only call `useCallStore.getState().setSpeakingPeers(newMap)` when any player's speaking status actually CHANGES compared to a ref tracking previous values. This prevents re-render storms.
- Use a `useRef` to store the previous speaking map for comparison.
- Handle `AudioContext` suspended state: after creation, if `audioCtx.state === 'suspended'`, call `audioCtx.resume()`.
- Cleanup: `cancelAnimationFrame`, disconnect all sources, close AudioContext.
- IMPORTANT: Use a stable ref for streams to avoid recreating the AudioContext on every render. Use `useRef` to track previous stream identities (by stream.id) and only rebuild analysers when streams actually change.
