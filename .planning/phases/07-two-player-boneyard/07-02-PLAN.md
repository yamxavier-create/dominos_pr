In `processAutoPassCascade`, BEFORE the existing "Auto-pass this player" block, add a boneyard draw phase. The logic:

```typescript
// BONEYARD DRAW PHASE (only when boneyard has tiles)
while (game.boneyard.length > 0) {
  const checkPlays = getValidPlays(
    game.players[idx].tiles,
    game.board,
    game.firstPlayMade,
    game.forcedFirstTileId
  )
  if (checkPlays.length > 0) break  // player can play with current hand

  // Draw one tile from boneyard
  const drawnTile = game.boneyard.pop()!
  game.players[idx].tiles.push(drawnTile)

  // Emit draw event to drawing player (with tile data)
  const drawingSocket = game.players[idx].socketId
  io.to(drawingSocket).emit('game:boneyard_draw', {
    tile: drawnTile,
    boneyardRemaining: game.boneyard.length,
    playerIndex: idx,
  })

  // Emit draw event to all others (without tile data)
  io.to(game.roomCode).except(drawingSocket).emit('game:boneyard_draw', {
    tile: null,
    boneyardRemaining: game.boneyard.length,
    playerIndex: idx,
  })
}
```

Place this BEFORE the existing `const validPlays = getValidPlays(...)` check. After the boneyard draw loop, the existing valid plays check will re-evaluate with the updated hand (which may now include drawn tiles).

IMPORTANT: The boneyard draw loop must come BEFORE the valid plays check that decides whether to auto-pass. The flow is:
1. Draw from boneyard until playable or empty
2. Check valid plays (now with potentially larger hand)
3. If valid plays exist -> it's their turn (return false)
4. If no valid plays -> auto-pass (existing logic)

Do NOT emit a separate state snapshot after each draw -- the individual `game:boneyard_draw` events handle incremental updates. A full `broadcastState` happens after the cascade resolves.

Note: `io.to(socketId).except(socketId)` requires Socket.io v4. Use the pattern shown above: emit to the specific player socket, then emit to the room excluding that socket. Use `io.to(game.roomCode).except(drawingSocket)` which is the standard Socket.io API.
