  <files>server/src/game/GameState.ts, server/src/socket/chatHandlers.ts, server/src/socket/handlers.ts</files>
  <action>
1. In `server/src/game/GameState.ts`:
   - Add `ChatMessage` interface: `{ id: string, playerIndex: number, playerName: string, content: string, type: 'text' | 'reaction', timestamp: number }`
   - Add `chatHistory: ChatMessage[]` to the `Room` interface (default `[]`)
   - Export `ChatMessage`

2. Create `server/src/socket/chatHandlers.ts` with `registerChatHandlers(socket, io, rooms)`:
   - Define `QUICK_REACTIONS = ['¡Capicú!', '¡Trancado!', '¡Buena jugada!', '¡Mala suerte!', '🔥', '🤡']`
   - Rate limiting: `Map<string, number[]>` keyed by socket.id. On each `chat:send`, filter timestamps to last 10s, reject if >= 5. Clean up on `disconnect`.
   - Sanitization: `content.replace(/<[^>]*>/g, '').trim().slice(0, 200)`. Reject empty after sanitization.
   - For `type: 'reaction'`, validate content is in QUICK_REACTIONS allowlist.
   - Derive `playerName` from room state (not client payload) to prevent spoofing.
   - Generate `id` via `crypto.randomUUID()`.
   - Push to `room.chatHistory`, cap at 50 messages (`.slice(-50)`).
   - Broadcast: `io.to(roomCode).emit('chat:message', chatMessage)`.
   - On `disconnect`: delete rate limit entry for socket.id.

3. In `server/src/socket/handlers.ts`:
   - Import and call `registerChatHandlers(socket, io, rooms)`.

4. In `server/src/socket/roomHandlers.ts` (the `room:rejoin` handler):
   - After existing reconnect logic, if `room.chatHistory?.length`, emit `socket.emit('chat:history', { messages: room.chatHistory })`.

5. Clear `room.chatHistory = []` in server `game:start` handler (same location that resets scores) to prevent old messages bleeding into new games.
