  <name>Task 1: Install vite-plugin-pwa, create icons, configure manifest and service worker</name>
  <files>client/vite.config.ts, client/index.html, client/tsconfig.json, client/public/pwa-192x192.png, client/public/pwa-512x512.png, client/public/pwa-maskable-192x192.png, client/public/pwa-maskable-512x512.png</files>
  <action>
1. Install vite-plugin-pwa:
   ```bash
   npm install -D vite-plugin-pwa --workspace=client
   ```

2. Create `client/public/` directory and generate 4 icon PNG files. Use a simple domino-themed design: a dark green (#0A1A0F) background with a white domino tile shape and "PR" text. Generate icons programmatically using a canvas-based approach or sharp/jimp library, or create simple solid-color placeholder PNGs with ImageMagick (`convert`). Required files:
   - `pwa-192x192.png` (192x192, standard icon)
   - `pwa-512x512.png` (512x512, standard icon)
   - `pwa-maskable-192x192.png` (192x192, maskable -- must have ~20% safe zone padding around the design so content is not clipped on circular/squircle masks)
   - `pwa-maskable-512x512.png` (512x512, maskable -- same safe zone padding)

   If ImageMagick is not available, use Node.js with the `canvas` package or write a simple script. The icons should be identifiable as Domino PR, not blank squares.

3. Update `client/vite.config.ts` -- add VitePWA plugin with the exact configuration from RESEARCH.md:
   - Import `{ VitePWA }` from `'vite-plugin-pwa'`
   - Add VitePWA to plugins array AFTER react()
   - `registerType: 'autoUpdate'`
   - Manifest: name "Domino PR", short_name "Domino PR", description "Domino puertorriqueno online con amigos", theme_color "#0A1A0F", background_color "#0A1A0F", display "standalone", orientation "portrait", scope "/", start_url "/"
   - Icons array: 4 entries (192 standard, 512 standard, 192 maskable, 512 maskable). IMPORTANT: Do NOT use `"any maskable"` combined purpose -- separate entries for each.
   - Workbox config: globPatterns for js/css/html/ico/png/svg/woff2, navigateFallback "index.html", navigateFallbackDenylist with `/^\/socket\.io/` and `/^\/health/` patterns
   - Runtime caching: CacheFirst for Google Fonts (googleapis.com and gstatic.com) with 1-year expiration
   - Keep existing server config (port, host, allowedHosts, proxy) unchanged

4. Update `client/index.html` -- add two tags in `<head>` (after the existing meta tags, before the Google Fonts preconnect):
   - `<meta name="theme-color" content="#0A1A0F" />`
   - `<link rel="apple-touch-icon" href="/pwa-192x192.png" />`
   Note: Do NOT manually add a manifest link -- vite-plugin-pwa auto-injects it.

5. Update `client/tsconfig.json` -- add `"types": ["vite-plugin-pwa/client"]` to compilerOptions so TypeScript resolves the `virtual:pwa-register` module.
