<objective>
Make the existing WebRTC video/audio call system player-count-aware so it works correctly in 2-player (duo) mode.

Purpose: Phase 5 built WebRTC for 4-player mode with hardcoded `i < 4` loops and `% 4` arithmetic. In 2-player games, this creates phantom peer connections for non-existent players 2 and 3, wasting resources and causing 15-second timeout failures. This plan replaces all 4 hardcoded assumptions with dynamic player count logic.

Output: Two modified files — `useWebRTC.ts` and `VideoCallPanel.tsx` — with zero new dependencies, zero server changes, and zero new files.
</objective>
