Create a reusable `requireAuth` helper at the top of the file (NOT middleware, inline helper like authRoutes pattern):
```typescript
import { Router, Request, Response } from 'express'
import prisma from '../db/prisma'
import { verifyToken } from '../auth/jwt'

const router = Router()

async function requireAuth(req: Request, res: Response): Promise<string | null> {
  const authHeader = req.headers.authorization
  if (!authHeader?.startsWith('Bearer ')) {
    res.status(401).json({ error: 'Authentication required' })
    return null
  }
  try {
    const payload = verifyToken(authHeader.slice(7))
    return payload.sub
  } catch {
    res.status(401).json({ error: 'Invalid token' })
    return null
  }
}
```

**social:friend_request** (FRD-02) -- accepts `{ targetUserId: string }`:
1. Guard: auth check, self-friend check (`userId === targetUserId` -> error "Cannot friend yourself")
2. Rate limit: `prisma.friendship.count({ where: { requesterId: userId, status: 'PENDING' } })` >= 20 -> error "Too many pending requests (max 20)"
3. CRITICAL bidirectional check: `prisma.friendship.findFirst({ where: { OR: [{ requesterId: userId, targetId: targetUserId }, { requesterId: targetUserId, targetId: userId }] } })`
4. If existing with status ACCEPTED -> error "Already friends"
5. If existing with `requesterId === userId` and PENDING -> error "Request already sent"
6. If existing with `requesterId === targetUserId` and PENDING -> AUTO-ACCEPT: `prisma.friendship.update({ where: { id: existing.id }, data: { status: 'ACCEPTED' } })` with include on both requester and target (select: id, username, displayName, avatarUrl). Emit `social:friend_accepted` to both `socket` and `io.to('user:' + targetUserId)` with `{ friendshipId, friend: <the other user> }`
7. Otherwise create new: `prisma.friendship.create({ data: { requesterId: userId, targetId: targetUserId } })` with include on requester and target. Emit `social:friend_request_sent` to socket with `{ requestId: friendship.id, to: friendship.target }`. Emit `social:friend_request_received` to `io.to('user:' + targetUserId)` with `{ requestId: friendship.id, from: friendship.requester }`
8. Wrap all in try/catch, emit `social:error` on failure

**social:friend_accept** (FRD-03) -- accepts `{ requestId: string }`:
1. Auth guard
2. Find friendship by id: `prisma.friendship.findUnique({ where: { id: requestId } })`
3. Validate: friendship exists, `friendship.targetId === userId` (only the target can accept), `friendship.status === 'PENDING'`
4. Update: `prisma.friendship.update({ where: { id: requestId }, data: { status: 'ACCEPTED' }, include: { requester: { select: ... }, target: { select: ... } } })`
5. Emit `social:friend_accepted` to socket with `{ friendshipId: updated.id, friend: updated.requester }` (the other user)
6. Emit `social:friend_accepted` to `io.to('user:' + updated.requesterId)` with `{ friendshipId: updated.id, friend: updated.target }` (us)

**social:friend_reject** (FRD-03) -- accepts `{ requestId: string }`:
1. Auth guard
2. Find friendship by id, validate `targetId === userId` and `status === 'PENDING'`
3. DELETE the row: `prisma.friendship.delete({ where: { id: requestId } })`
4. Emit `social:friend_rejected` to socket with `{ requestId }`
5. Emit `social:friend_rejected` to `io.to('user:' + friendship.requesterId)` with `{ requestId }`

**social:friend_remove** (FRD-05) -- accepts `{ friendUserId: string }`:
1. Auth guard
2. Delete bidirectionally: `prisma.friendship.deleteMany({ where: { status: 'ACCEPTED', OR: [{ requesterId: userId, targetId: friendUserId }, { requesterId: friendUserId, targetId: userId }] } })`
3. If `deleteCount === 0`, emit error "Not friends"
4. Emit `social:friend_removed` to socket with `{ userId: friendUserId }`
5. Emit `social:friend_removed` to `io.to('user:' + friendUserId)` with `{ userId }`

Close the function. All errors caught in try/catch with `social:error` emission.
