  <action>
Create `client/src/audio/` directory and two modules:

**audioContext.ts** -- Module-level singleton with autoplay unlock:
- Module-scoped `audioContext: AudioContext | null` and `unlockRegistered: boolean`
- `getAudioContext(): AudioContext` -- lazily creates one AudioContext on first call, then returns same instance forever. Calls `registerAutoplayUnlock()` on creation.
- `registerAutoplayUnlock(): void` -- registers ONE-TIME document-level listeners for `click`, `touchend`, `keydown` (all with `capture: true`). The handler checks `audioContext.state === 'suspended'` and calls `audioContext.resume()`. After successful resume (state becomes `running`), removes all three listeners. Uses a guard boolean to prevent double-registration.
- Do NOT export the raw AudioContext variable -- only the getter function. This ensures lazy initialization.

**audioLoader.ts** -- Buffer cache and playback:
- Module-scoped `bufferCache: Map<string, AudioBuffer>`
- `loadAudio(url: string): Promise<AudioBuffer>` -- returns cached buffer if available. Otherwise: calls `getAudioContext()`, ensures context is running via `ctx.resume()` (safety net for decodeAudioData on suspended contexts per research pitfall 4), fetches URL, decodes arrayBuffer via `ctx.decodeAudioData()`, caches result, returns buffer.
- `playBuffer(buffer: AudioBuffer, volume?: number): void` -- gets context via `getAudioContext()`, calls `ctx.resume()` as safety net (research pitfall 3), creates `BufferSourceNode`, connects through `GainNode` if volume !== 1.0, otherwise connects directly to `ctx.destination`, calls `source.start(0)`.
- Both functions import `getAudioContext` from `./audioContext`.

TypeScript strict mode must pass. No `any` types.
