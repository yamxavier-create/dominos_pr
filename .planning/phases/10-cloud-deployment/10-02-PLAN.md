  <action>
Create a step-by-step guide for configuring a custom domain on Railway. This satisfies DEPLOY-02 (app supports custom domain configuration). The guide should cover:

1. Prerequisites: App already deployed on Railway (from the checkpoint task below)
2. In Railway dashboard: navigate to Service -> Settings -> Domains -> Add Custom Domain
3. Enter the desired domain (e.g., dominospr.com or play.dominospr.com)
4. Railway provides a CNAME target (e.g., something.up.railway.app)
5. In DNS provider (Cloudflare, Namecheap, etc.): Create a CNAME record
   - For subdomain (play.example.com): Name = "play", Value = Railway CNAME target
   - For root domain (@): depends on provider — Cloudflare supports CNAME flattening, others may need ALIAS record
6. Wait for Railway to verify DNS (usually under 5 min, can take up to 72 hours)
7. SSL certificate auto-provisioned by Railway (Let's Encrypt) — no action needed
8. Verify by visiting the custom domain in browser

Include a troubleshooting section:
- DNS not propagated yet: use `dig CNAME yourdomain.com` to check
- SSL not ready: wait 5-10 min after DNS verification, Railway auto-provisions
- Redirect loop: ensure no conflicting A records for the domain

Note: The user does NOT need to do this during phase 10. This is documentation for when they have a domain ready. The Railway-provided *.up.railway.app URL is the permanent URL for DEPLOY-01.
