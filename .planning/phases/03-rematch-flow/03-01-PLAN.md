   - `RematchVoteUpdate { votes: number[]; playerNames: string[] }` — votes = array of playerIndex who voted, playerNames = all 4 player names in seat order
   - `RematchCancelled { disconnectedPlayerIndex: number; disconnectedPlayerName: string }`

3. In `server/src/socket/gameHandlers.ts`, add inside `registerGameHandlers`:

   a. `game:rematch_vote` handler:
      - Validate: room exists, room.game exists, game.phase === 'game_end'
      - Find player by socket.id in game.players
      - Initialize `room.rematchVotes` to `[]` if undefined
      - If player.index already in rematchVotes, return (no double vote)
      - Push player.index to room.rematchVotes
      - Broadcast `game:rematch_vote_update` to room with `{ votes: room.rematchVotes, playerNames: game.players.map(p => p.name) }`
      - If room.rematchVotes.length === 4:
        - Emit `game:rematch_accepted` to room (empty payload)
        - After 2000ms setTimeout: reset game exactly like `game:next_game` handler (reuse same logic — shuffle, deal, reset scores to 0-0, same seats, winner starts, emit `game:started` per player)
        - Clear room.rematchVotes to []

   b. On `disconnect` event (add near existing disconnect handling in the file or in the socket.on('disconnect') handler — check where disconnect is handled):
      - If room has active rematchVotes (length > 0) and disconnecting player was in game.players:
        - Emit `game:rematch_cancelled` to room with `{ disconnectedPlayerIndex, disconnectedPlayerName }`
        - Clear room.rematchVotes to []

   c. Initialize `rematchVotes: []` in `game:start` handler where room properties are set, and clear it in `game:next_game` handler.

   IMPORTANT: The disconnect handler may be in a separate file (handlers.ts or roomHandlers). Check `server/src/socket/handlers.ts` for where `disconnect` is handled. If it's there, add rematch cancellation logic there. If gameHandlers doesn't have access to it, export a helper function `checkRematchCancellation(io, room, disconnectedSocketId)` from gameHandlers that handlers.ts can call.
