### Wiring Gaps

**GAP-1: Host detection in RoomLobby.tsx (BUG-02)**
- `client/src/components/lobby/RoomLobby.tsx:18`: `const isHost = myPlayerIndex === 0`
- BUG-02 fixed host detection in modals using `hostSocketId`, but `RoomLobby` still uses seat index
- Currently incidentally correct (host promotion always picks seat-0 occupant), but violates the design intent of BUG-02
- Severity: low (no runtime breakage confirmed)

**GAP-2: Rate limit constant mismatch (CHAT-06)**
- `server/src/socket/chatHandlers.ts:7`: `RATE_LIMIT_MAX = 15`
- Requirement CHAT-06 specifies: 5 msgs / 10 seconds
- Actual limit is 3× the spec — would fail acceptance testing

### Broken Flows

**FLOW-1 & FLOW-2: Stale chat history on reconnect after rematch or next_game (CHAT-05)**
- `game:rematch_vote` consensus path does NOT clear `room.chatHistory`
- `game:next_game` path does NOT clear `room.chatHistory`
- `game:start` (initial) DOES clear `room.chatHistory` at line 341
- Client-side: `clearChatState()` IS called in `game:started` handler — active clients are fine
- Impact: a player who disconnects and reconnects after a rematch or next_game receives messages from the previous game via `chat:history`
- Fix: add `room.chatHistory = []` in `gameHandlers.ts` at the `game:next_game` path (~line 563) and the rematch consensus path (~line 620)
