## Tech Debt (3 items, 1 phase)

### Phase 12: Avatar Cameras

1. **AvatarVideo useEffect edge case** — `videoRef.current.srcObject = stream` runs in a `useEffect([stream])` but the `<video>` element only mounts when `showVideo` is true. If stream changes while `isCameraOff=true`, the assignment is skipped. When camera toggles back on, video may show blank until next stream change. Recoverable by re-toggling camera.

2. **Dead code: VideoCallPanel.tsx** — File still exists at `client/src/components/game/VideoCallPanel.tsx` but is not imported anywhere. Should be deleted to avoid confusion.

3. **JoinCallButton forces audio+video** — `joinCallRef.current?.(true, true)` always requests both audio and video. Players who want audio-only mid-game get an unexpected camera permission prompt. `useWebRTC.joinCall` has a fallback to audio-only if combined request fails, so it degrades gracefully but the UX is suboptimal.
